🚀 QUICK START (5 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Install Node.js
────────────────────────
1. Go to: https://nodejs.org/
2. Download LTS version (18.18.2 or higher)
3. Run the installer (accept all defaults)
4. Restart your terminal/PowerShell
5. Verify: Type "node --version" in terminal

STEP 2: Install Dependencies
────────────────────────────────
Windows (PowerShell):
    cd C:\CRM\backend
    npm install
    cd ..\frontend
    npm install
    cd ..

macOS/Linux:
    cd /path/to/CRM/backend
    npm install
    cd ../frontend
    npm install
    cd ..

STEP 3: Install Docker
──────────────────────
1. Go to: https://www.docker.com/products/docker-desktop
2. Download and install Docker Desktop
3. Restart your computer

STEP 4: Start MySQL
──────────────────
Open terminal and run:
    cd C:\CRM
    docker-compose up -d

Wait 10 seconds for MySQL to start.
You should see: "crm_mysql is running"

STEP 5: Start Backend Server
────────────────────────────
Open NEW terminal:
    cd C:\CRM\backend
    npm run dev

Wait for message:
    ✓ Database connected
    ✓ Models synced
    ✓ Server running on http://localhost:3000

STEP 6: Start Frontend Server
─────────────────────────────
Open ANOTHER new terminal:
    cd C:\CRM\frontend
    npm run dev

Wait for message:
    VITE ready in ... ms
    Local: http://localhost:5173/

STEP 7: Open Browser
────────────────────
Go to: http://localhost:5173

Login with:
    Email: admin@crm.com
    Password: admin123

✨ You're in! ✨

🔍 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: "npm: command not found"
→ Solution: Node.js not installed. Download from https://nodejs.org/

Problem: "Cannot connect to MySQL"
→ Solution: Run "docker-compose up -d" in C:\CRM directory

Problem: "Cannot GET /people"
→ Solution: Backend not running. Run "npm run dev" in backend folder

Problem: "Network Error" when logging in
→ Solution: Frontend/Backend not communicating. Check both servers running.

Problem: "Port already in use"
→ Solution: Another app is using the port. 
   - Close other terminals
   - Or change PORT in backend/.env to different number
   - Or restart computer

For more help, see: SETUP.md
